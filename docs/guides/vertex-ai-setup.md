# Setting Up Vertex AI Authentication for AI Tools in DevWorkspaces

This guide explains how to configure Google Cloud Vertex AI credentials for AI CLI tools (Claude Code, OpenCode, Gemini CLI) running inside Eclipse Che DevWorkspaces.

## Prerequisites

- `gcloud` CLI installed locally
- Access to a Google Cloud project with Vertex AI enabled
- `kubectl` access to the Che cluster namespace

## Step 1: Generate Application Default Credentials

On your local machine, run:

```sh
gcloud auth application-default login
gcloud auth application-default set-quota-project cloudability-it-gemini
```

This opens a browser for authentication and saves credentials to `~/.config/gcloud/application_default_credentials.json`.

## Step 2: Create a Kubernetes Secret

Create a secret from the ADC file in your Che namespace:

```sh
kubectl create secret generic gcloud-adc \
  -n <your-namespace> \
  --from-file=adc.json=$HOME/.config/gcloud/application_default_credentials.json
```

## Step 3: Label the Secret for Automount

Label and annotate the secret so the DevWorkspace Operator automatically mounts it into every workspace:

```sh
kubectl label secret gcloud-adc -n <your-namespace> \
  controller.devfile.io/mount-to-devworkspace=true \
  controller.devfile.io/watch-secret=true \
  --overwrite

kubectl annotate secret gcloud-adc -n <your-namespace> \
  controller.devfile.io/mount-path=/credentials \
  controller.devfile.io/mount-as=file \
  --overwrite
```

After this, the file `/credentials/adc.json` will be available in every new or restarted workspace in your namespace.

## Step 4: Set Environment Variables

Set the following environment variables in your workspace. You can do this via a ConfigMap with DWO automount, or by adding them to your DevWorkspace spec.

### Claude Code

```sh
GOOGLE_APPLICATION_CREDENTIALS=/credentials/adc.json
CLAUDE_CODE_USE_VERTEX=1
ANTHROPIC_VERTEX_PROJECT_ID=<your-gcp-project-id>
ANTHROPIC_VERTEX_REGION=<your-region>
CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1
```

### OpenCode

```sh
GOOGLE_APPLICATION_CREDENTIALS=/credentials/adc.json
GOOGLE_CLOUD_PROJECT=<your-gcp-project-id>
VERTEX_LOCATION=<your-region>
```

### Gemini CLI

```sh
GOOGLE_APPLICATION_CREDENTIALS=/credentials/adc.json
GEMINI_CLI_HOME=/tmp/gemini-home
```

### Using a ConfigMap for Environment Variables

To set env vars automatically for all workspaces in your namespace:

```sh
kubectl create configmap ai-tools-env \
  -n <your-namespace> \
  --from-literal=GOOGLE_APPLICATION_CREDENTIALS=/credentials/adc.json \
  --from-literal=CLAUDE_CODE_USE_VERTEX=1 \
  --from-literal=ANTHROPIC_VERTEX_PROJECT_ID=<your-gcp-project-id> \
  --from-literal=ANTHROPIC_VERTEX_REGION=<your-region> \
  --from-literal=CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1 \
  --dry-run=client -o yaml | kubectl apply -f -

kubectl label configmap ai-tools-env -n <your-namespace> \
  controller.devfile.io/mount-to-devworkspace=true \
  controller.devfile.io/watch-configmap=true \
  --overwrite

kubectl annotate configmap ai-tools-env -n <your-namespace> \
  controller.devfile.io/mount-as=env \
  --overwrite
```

## Step 5: Restart Your Workspace

Restart any running workspaces to pick up the new secret and environment variables.

## Refreshing Credentials

ADC tokens generated by `gcloud auth application-default login` are refresh tokens, so they do not expire after a fixed period. However, if your organization rotates credentials or revokes access, you will need to:

1. Re-run `gcloud auth application-default login` locally
2. Update the secret:
   ```sh
   kubectl create secret generic gcloud-adc \
     -n <your-namespace> \
     --from-file=adc.json=$HOME/.config/gcloud/application_default_credentials.json \
     --dry-run=client -o yaml | kubectl apply -f -
   ```
3. Restart your workspace

## Troubleshooting

- **"file does not exist" error**: The secret is not mounted. Verify labels/annotations:
  ```sh
  kubectl describe secret gcloud-adc -n <your-namespace>
  ```
- **Authentication failed**: Re-run `gcloud auth application-default login` and update the secret
- **Wrong project/region**: Verify `ANTHROPIC_VERTEX_PROJECT_ID` and `ANTHROPIC_VERTEX_REGION` match your Vertex AI configuration
